# 💳 PaySlip — سامانهٔ محاسبه‌ی حقوق و دستمزد برای کسب‌وکارهای کوچک


## چالش

نرم‌افزارهای حسابداری سنتی مانند **هلو** یا **سپیدار**، برای کسب‌وکارهای کوچک مانند رستوران‌ها، فروشگاه‌ها یا دفاتر خدماتی، بسیار سنگین، پیچیده و گران‌قیمت هستند. این کسب‌وکارها نیازی به انبارداری یا حسابداری کامل ندارند؛ آنها تنها به **محاسبه‌ی دقیق و سریع حقوق کارمندان** خود نیازمندند.


## راه‌کار

 یک سیستم اختصاصی است که صرفاً بر روی محاسبه‌ی حقوق و دستمزد متمرکز شده و این کار را به بهترین شکل ممکن انجام می‌دهد

- ✅ کارفرما به‌راحتی پرسنل خود را تعریف می‌کند.
- ✅ اطلاعات ماهیانه (غیبت، اضافه‌کاری، مرخصی و ...) به‌سادگی ثبت می‌شود.
- ✅ موتور محاسباتی سیستم، فیش حقوقی را به‌صورت خودکار محاسبه کرده و خروجی **PDF** تحویل می‌دهد.
- ✅ با تغییر قوانین کار در هر سال، سیستم **بدون نیاز به تغییر در کد** و صرفاً با به‌روزرسانی پیکربندی، خود را تطبیق می‌دهد.


## 🧠 چالش‌های فنی پیاده‌سازی‌شده

### ۱. موتور قوانین (Rule Engine)
قوانین حقوق و دستمزد (درصد بیمه، مالیات، عیدی، سنوات و ...) کاملاً از هسته‌ی کد جدا شده‌اند. به محض اعلام تغییرات سالانه از سوی وزارت کار، تنها فایل پیکربندی مربوطه آپدیت می‌شود — بدون نیاز به دیپلوی مجدد.

### ۲. چندمستأجری (Multi-tenancy)
هر کسب‌وکار به‌عنوان یک **Tenant** مستقل در نظر گرفته می‌شود. داده‌های مالی و پرسنلی هر شرکت کاملاً از سایرین ایزوله است و هیچ داده‌ای بین آن‌ها نشت نمی‌کند.

### ۳. پردازش‌های زمان‌بندی‌شده (Background Jobs)
در پایان هر ماه، سیستم به‌صورت خودکار و در پس‌زمینه، فیش حقوقی تمامی کارمندان را محاسبه و تولید (Generate) می‌کند.

### ۴. گزارش‌گیری از تغییرات (Audit Log)
هرگونه تغییر در مبلغ حقوق یا اطلاعات پرسنلی، همراه با **زمان (Timestamp)** و **دلیل** ثبت می‌شود. اگر کارمندی نسبت به فیش خود اعتراضی داشته باشد، تاریخچه‌ی کاملی برای بررسی موجود است.

### ۵. معماری رویدادمحور (Event-driven)
ثبت هر رویداد مانند غیبت یا اضافه‌کاری، یک **Event** در سیستم صادر می‌کند که به‌صورت خودکار روی محاسبات پایان‌ماه تأثیر می‌گذارد و فرآیندها را واکنش‌گرا می‌سازد.


## 🛠 فناوری‌های استفاده‌شده

| لایه (لایه) | ابزار (ابزار) |
|---|---|
| هسته‌ی بک‌اند | ASP.NET Core ۱۰، Minimal API |
| معماری | Modular Monolith، CQRS، MediatR |
| پایگاه داده | SQL Server، EF Core |
| پردازش‌های زمان‌بندی | BackgroundService، Hangfire |
| احراز هویت | JWT، ASP.NET Identity |
| رابط کاربری | Blazor WebAssembly |


## 📂 ساختار ماژول‌ها

```
PaySlip/
├── Modules/
│   ├── Identity/         ← احراز هویت و مدیریت Tenant
│   ├── Employee/         ← مدیریت اطلاعات کارمندان
│   ├── Attendance/       ← ثبت حضور، غیبت، اضافه‌کاری و مرخصی
│   ├── Payroll/          ← محاسبه‌ی حقوق و تولید فیش PDF
│   └── Rules/            ← موتور قوانین (قابل تغییر توسط پیکربندی)
├── Shared/
│   ├── Kernel/           ← الگوی Result، موجودیت‌های پایه
│   ├── Contracts/        ← Commands، Queries و Events
│   └── Infrastructure/   ← گزارش‌گیری (Audit)، میان‌افزار Multi-tenancy
└── API/                  ← نقطه‌ی ورود برنامه
```
